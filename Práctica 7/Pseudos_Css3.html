PSEUDOCLASES

:active

La pseudoclase :active representa a un elemento que el usuario ha activado, generalmente, haciendo clic sobre el elemento con el botón izquierdo del ratón. Se suele utilizar en botones y enlaces.

Sintaxis

elemento:active {
    /* Definición de estilos */
}

Los estilos definidos por :active pueden ser anulados por las posteriores definiciones de :link, :hover o :visited. Es una buena práctica colocar :active después de todas las definiciones de estilo de los enlaces.

Ejemplos:

button:active {
    color: red;
    background-color: black;
}
                
a:active {
    background-color: yellow;
    text-decoration: none;
}
Al pinchar en el enlace, se pondrá sobre fondo amarillo y no tendrá subrayado

:any-link

La pseudoclase :any-link de CSS representa a un elemento que actúa como el ancla origen de un hipervínculo independientemente de si ha sido visitado, es decir, coincide con cualquier elemento <a>, <area> o <link> con un atributo href. Por lo tanto, coincide con todos los elementos que coincidan con :link o :visited.

Sintaxis

elemento:any-link {
    /* Definición de estilos */
}

Los estilos definidos por :active pueden ser anulados por las posteriores definiciones de :link, :hover o :visited. Es una buena práctica colocar :active después de todas las definiciones de estilo de los enlaces.

Ejemplos:

a:any-link {
    border: 1px solid blue;
    color: orange;
}

:checked

La pseudoclase :checked se usa principalmente con elementos de formulario, como casillas de verificación y botones de opción.

Sintaxis

input[type="checkbox"]:checked {
    /* Definición de estilos */
}

El usuario puede activar este estado marcando/seleccionando un elemento, o desactivándolo desmarcando el elemento.

Ejemplos:

input[type="checkbox"]:checked {
    margin: 50px
}

:current

La pseudoclase :current se utiliza en el contexto de navegación y se aplica a un enlace <a> que representa la página actual. Es decir, selecciona el enlace que corresponde a la página en la que te encuentras actualmente.

Sintaxis

a:current {
    /* Definición de estilos */
}

Ejemplos:

a:current {
    color: red;
    font-weight: bold;
}

:default

La pseudoclase :default de CSS representa cualquier elemento de formulario que sea el predeterminado entre un grupo de elementos relacionados.

Sintaxis

input:default {
    /* Definición de estilos */
}

Los elementos agrupados que permiten selecciones múltiples también pueden tener múltiples valores predeterminados, es decir, pueden tener múltiples elementos seleccionados inicialmente.

Ejemplos:

input:default {
    box-shadow: 0 0 2px 1px coral;
}

:disabled

La pseudoclase :disabled de CSS representa a cualquier elemento deshabilitado. Un elemento se encuentra deshabilitado si no puede ser activado (por ejemplo ser selecionado, hacer click en él o aceptar texto de entrada) ni aceptar el foco.

Sintaxis

input:disabled {
    /* Definición de estilos */
}

Ejemplos:

input:disabled {
    background-color: #ccc;
}

:empty

La pseudo-clase :empty de CSS representa cualquier elemento que no tenga hijos. Los hijos pueden ser nodos de elemento o texto (incluido el espacio en blanco). Los comentarios o las instrucciones de procesamiento no afectan si un elemento se considera vacío.

Sintaxis

elemento:empty {
    /* Definición de estilos */
}

Ejemplos:

elemento:empty {
    background: lime;
}
Voy a ser de color rosa.

:enabled

Sintaxis

input:enabled {
    /* Definición de estilos */
}

Ejemplos:

input:enabled {
    background: lime;
}

:focus

La pseudoclase :focus CSS representa un elemento (como una entrada de formulario) que ha recibido el foco. Generalmente se activa cuando el usuario hace clic, toca un elemento o lo selecciona con la tecla "Tab" del teclado.

Sintaxis

input:focus {
    /* Definición de estilos */
}

Ejemplos:

input:focus {
    border-color: blue;
    box-shadow: 0 0 5px rgba(0, 0, 255, 0.5);
}

button:focus {
    outline: 4px dashed darkorange;
}

:focus-visible

La pseudoclase :focus-visible se aplica mientras un elemento coincide con la pseudo-clase :focus. Este selector es útil para proporcionar un indicador de enfoque diferente basado en la modalidad de entrada del usuario (ratón vs teclado).

Sintaxis

input:focus-visible {
    /* Definición de estilos */
}

Ejemplos:

input:focus-visible {
    outline: 2px solid green;
    background-color: #f0f0f0;
}


:focus-within

La pseudoclase CSS :focus-within representa un elemento que ha recibido el foco o que contiene un elemento que ha recibido el foco. En otras palabras, representa un elemento que en sí coincide con la pseudoclase :focus o tiene descendientes que coincidan con :focus.

Sintaxis

form:focus-within {
    /* Definición de estilos */
}

Este selector es útil, por tomar un ejemplo común, para resaltar un contenedor <form> completo cuando el usuario enfoca sobre uno de sus elementos <input>.

Ejemplos:

form:focus-within {
    background: #ff8;
    color: black;
}

:hover

La pseudoclase :hover en CSS se utiliza para seleccionar y aplicar estilos a un elemento cuando el usuario coloca el cursor sobre él. Es comúnmente utilizado para proporcionar efectos visuales y de interactividad, como cambiar el color de fondo, resaltar bordes, mostrar información adicional, etc.

Sintaxis

elemento:hover {
    /* Definición de estilos */
}

Ejemplos:

elemento:hover {
    background-color: gold;
}

:in-range

Selecciona un elemento (input) cuyo valor se encuentra dentro de un rango de valores determinado.

Sintaxis

input:in-range [Aqui va la sintaxis de cada propiedad];

Ejemplos:

input:in-range {
                background-color: rgba(0, 255, 0, 0.25);
            }
Obtenemos aquí un cambio de CSS al estar el valor dentro de un rango determinado en el HTML. Esta propiedad nos ahorra mucho código JS.

:placeholder-shown

La pseudo-clase :placeholder-shown de CSS representa cualquier elemento <input> o <textarea> que esté mostrando actualmente el texto de marcador de posición (placeholder).

Sintaxis

/* Selecciona cualquier elemento con un placeholder activo */
:placeholder-shown {
    /* ... */
}

Ejemplos:

input {
    border: 2px solid black;
    padding: 3px;
}

input:placeholder-shown {
    border-color: silver;
}

En pantallas angostas como teléfonos inteligentes, el ancho de los cuadros de búsqueda y otros campos de formulario pueden acortarse drásticamente. Esto puede provocar que el texto de marcador de posición se recorte de una manera no deseada. A menudo es útil alterar este comportamiento con la propiedad text-overflow. 

input:placeholder-shown {
text-overflow: ellipsis;
}

:playing

La pseudoclase :playing en CSS se utiliza para seleccionar y aplicar estilos a elementos reproducibles, como audio o video, que se encuentran en estado de reproducción. Un elemento se considera que está "playing" si actualmente está reproduciendo un recurso multimedia, o si ha detenido temporalmente la reproducción por razones que no son la intención del usuario, como la carga de datos (buffering) o una parada (stalled)

Sintaxis

:playing {
/* propiedades CSS */
}

Ejemplos:

:playing {
  border: 5px solid green;
}
Este código agregaría un borde sólido verde de 5 píxeles a los elementos de video mientras se están reproduciendo

:read-only

La pseudoclase :read-only de CSS se utiliza para representar elementos no editables por el usuario, como input o textarea. Estos elementos son aquellos que no pueden ser modificados directamente, como un campo de un formulario que se presenta solo para lectura.

Sintaxis

:read-only {
/* propiedades CSS */
}

Ejemplos:

Para formularios: Cuando se quiere que el usuario verifique la información ingresada en un formulario sin poder modificarla, se podría utilizar :read-only para estilizar esos elementos de manera que no parezcan campos clickeables: 
input:read-only,
textarea:read-only {
  border: 0;
  box-shadow: none;
  background-color: white;
}

Para controles no formales:

Además de <input> y <textarea>, :read-only también puede seleccionar cualquier elemento que no sea editable: reproduciendo 
p:read-only {
  background-color: red;
  color: white;
}
Este último ejemplo haría que un párrafo no editable tenga un fondo rojo y texto blanco

:read-write

La pseudoclase :read-write de CSS selecciona elementos que son editables por el usuario, como <input> o <textarea>. Esto permite aplicar estilos específicos a elementos de formulario que el usuario puede modificar.

Sintaxis

:read-write {
  /* propiedades CSS */
}

Ejemplos:

Para controles de formulario: Podrías querer aplicar estilos distintos a un <textarea> que es editable, resaltándolo para que el usuario sepa que puede interactuar con él: 
textarea:read-write {
  box-shadow: inset 1px 1px 3px #ccc;
  border-radius: 5px;
}
Para controles no formales: La pseudoclase :read-write también puede seleccionar cualquier elemento editable, como un párrafo con el atributo contenteditable: 
p:read-write {
  background-color: lime;
}
Estos ejemplos aplicarían un sombreado interior y bordes redondeados al <textarea> editable, y un fondo color lima a un párrafo editable, respectivamente

:required

La pseudoclase :required en CSS se utiliza para seleccionar elementos de formulario como <input>, select, o <textarea> que tienen el atributo required. Este atributo se utiliza para indicar que un campo debe ser llenado antes de que el formulario pueda ser enviado. La pseudoclase es útil para resaltar estos campos obligatorios y asegurarse de que los usuarios no pasen por alto la necesidad de completarlos.

Sintaxis

:required {
  /* propiedades CSS */
}

Ejemplos:

<form>
  <label for="email_input">Enter an email address:
  <input type="email" id="email_input" required />
</form>

input:required {
  border-color: #800000; /* Dark red */
  border-width: 3px;
}

input:required:invalid {
  border-color: #c00000; /* Brighter red */
}
En este ejemplo, todos los campos de entrada (<input>) que son obligatorios tienen un borde de color rojo oscuro. Si el campo es obligatorio y contiene un valor inválido, el borde se cambia a un rojo más brillante para resaltar la necesidad de corrección​​. Es importante considerar las cuestiones de accesibilidad al usar el atributo required. Por ejemplo, las personas que utilizan tecnologías de asistencia como lectores de pantalla deben poder identificar qué campos son obligatorios. Por ello, además del uso de color, es recomendable emplear texto descriptivo o iconos para indicar la obligatoriedad del campo

:right

La pseudoclase :right en CSS se utiliza junto con la regla @page para representar todas las páginas derechas de un documento impreso. Esto puede ser útil cuando se están definiendo estilos para la impresión de documentos y se desea aplicar estilos específicos a las páginas que estarán en el lado derecho del documento impreso.

Sintaxis

@page :right {
  /* propiedades CSS */
}
Para seleccionar todas las páginas derechas cuando se imprime un documento y definir sus márgenes, se puede utilizar el siguiente código:

Ejemplos:

@page :right {
  margin: 2in 3in;
}
Si la dirección de escritura principal del documento es de izquierda a derecha, entonces la primera página será una página :right. Si la dirección es de derecha a izquierda, entonces la primera página será una página :left. Es importante señalar que esta pseudoclase solo puede usarse para cambiar las propiedades de margen, relleno, borde y fondo del cuadro de la página. Todas las demás propiedades serán ignoradas y solo el cuadro de la página, no el contenido del documento en la página, se verá afectado​

:root

La pseudoclase :root en CSS se utiliza para seleccionar el elemento raíz del documento, que es el elemento <html> en documentos HTML. Es útil para declarar variables CSS globales o para aplicar estilos que se quieren asegurar que sean de mayor especificidad que otros selectores.

Sintaxis

:root {
  /* propiedades CSS */
}
Puedes definir variables CSS que estén disponibles globalmente en tu documento de la siguiente manera:

Ejemplos:

:root {
  --color-principal: #333;
  --espaciado-interior: 1rem;
}

/* Usar las variables en otro selector */
body {
  color: var(--color-principal);
  padding: var(--espaciado-interior);
}
En este ejemplo, --color-principal y --espaciado-interior son variables CSS definidas en el elemento raíz y luego utilizadas en el cuerpo del documento. La pseudoclase :root tiene la especificidad más alta para el elemento <html>, por lo que los estilos definidos aquí superarán a los estilos definidos directamente en el elemento <html> sin esta pseudoclase.

:scope

La pseudoclase :scope en CSS se utiliza para referenciar elementos que son un punto de referencia o alcance para que los selectores coincidan con ellos. Es especialmente útil en el contexto de las APIs del DOM, como querySelector(), querySelectorAll(), matches(), o Element.closest(), donde :scope se refiere al elemento en el que se llamó al método.

Sintaxis

:scope {
  /* propiedades CSS */
}

Ejemplos:

Match de identidad usando JavaScript: 
const paragraph = document.getElementById("para");
const output = document.getElementById("output");

if (paragraph.matches(":scope")) {
  output.textContent = "The first paragraph is its own scope, as expected!";
}
Seleccionar hijos directos: 
const context = document.getElementById("context");
const selected = context.querySelectorAll(":scope > div");
En el primer ejemplo, se utiliza :scope para verificar si un párrafo es su propio ámbito, mientras que en el segundo, se seleccionan los hijos directos de un elemento ya recuperado. A nivel de hoja de estilos, :scope es equivalente a :root, ya que actualmente no hay una forma de establecer explícitamente un elemento de ámbito

:valid

La pseudoclase :valid en CSS se usa para seleccionar cualquier elemento <input> u otro elemento de formulario (<form>) cuyo contenido pasa con éxito las restricciones de validación de HTML. Es muy útil para resaltar campos de formulario que contienen datos correctamente formateados según las reglas de validación del formulario. Esta retroalimentación visual puede ayudar a los usuarios a confirmar que sus datos están formateados correctamente.

Sintaxis

:valid {
                        /* propiedades CSS */
                    }
Para indicar visualmente que un campo de formulario es válido, puedes usar la pseudoclase :valid junto con ::before o ::after para añadir iconos o indicadores:

Ejemplos:

input:valid + span::before {
  content: "✓";
  color: green;
}
Este código muestra un icono de verificación verde al lado de un campo de entrada válido.

:target

La pseudoclase :target en CSS se utiliza para estilizar un elemento único (el elemento objetivo) que tiene un ID que coincide con el fragmento de la URL.

Sintaxis

:target {
  /* propiedades CSS */
}

Ejemplos:

Si tienes una URL como http://www.example.com/index.html#section2, el siguiente elemento HTML sería seleccionado por el selector :target cuando la URL actual coincida con la anterior: 
<section id="section2">Ejemplo</section>
En CSS, podrías resaltar este elemento con un borde sólido: 
:target {
    border: 2px solid black;
  }
Este código agregaría un borde sólido verde de 5 píxeles a los elementos de video mientras se están reproduciendo

:visited

La pseudoclase :visited en CSS se aplica a los enlaces que el usuario ya ha visitado. Por razones de privacidad, los estilos que se pueden modificar utilizando este selector son muy limitados. Se aplica solo a los elementos y que tienen un atributo href.

Sintaxis

:visited {
  /* propiedades CSS */
}

Ejemplos:

a {
  background-color: white;
  border: 1px solid white;
}

a:visited {
  background-color: yellow;
  border-color: hotpink;
  color: hotpink;
}
En este ejemplo, los enlaces visitados se mostrarán con un fondo amarillo, un borde fucsia y texto del mismo color. Las propiedades que se pueden modificar con :visited incluyen color, background-color, border-color, etc., pero las restricciones de privacidad impiden cambios en otros estilos. Además, las funciones de CSS que pueden revelar si un enlace ha sido visitado (como window.getComputedStyle) mostrarán los valores de estilo del estado no visitado en lugar de los reales para :visited

PSEUDOELEMENTOS

::after

El pseudo-elemento ::after en CSS se utiliza para crear un pseudo-elemento que es el último hijo del elemento seleccionado. Se emplea frecuentemente para agregar contenido cosmético a un elemento usando la propiedad content. Por defecto, es en línea.

Sintaxis

::after {
  content: "texto o imagen";
  /* Otras propiedades CSS */
}

Ejemplos:

Uso simple: 
.exciting-text::after {
  content: " EXCITING!";
  color: green;
}
Ejemplo decorativo: 
.ribbon::after {
  content: "This is a fancy orange box.";
  background-color: #ffba10;
  border-color: black;
  border-style: dotted;
}
Si no se especifica la propiedad content, o si tiene un valor inválido, normal o none, entonces el pseudo-elemento ::after no se renderiza, como si tuviera display: none

::before

El pseudoelemento ::before en CSS se utiliza para insertar contenido antes del contenido de un elemento seleccionado. Es una forma de añadir contenido de forma dinámica con CSS sin necesidad de modificar el HTML. La sintaxis básica es la siguiente:

Sintaxis

selector::before {
  content: "texto o referencia a un contenido";
  /* Otros estilos */
}

Ejemplos:

Añadir texto antes de un elemento: 
p::before {
  content: "Nota: ";
  font-weight: bold;
}
Este código añadiría la palabra "Nota: " antes del contenido de todos los párrafos (<p>), y lo mostraría en negrita. Crear formas o decoraciones antes de un elemento: 
div::before {
  content: "";
  display: block;
  width: 50px;
  height: 50px;
  background-color: blue;
}
El pseudoelemento ::before es un recurso muy poderoso para diseñadores web que permite una gran flexibilidad sin alterar la estructura del HTML.

::first-letter

El pseudo-elemento ::first-letter en CSS se utiliza para aplicar estilos a la primera letra de la primera línea de un contenedor de bloque, siempre que no esté precedida por otro contenido, como imágenes o tablas en línea. Algunas de las propiedades de CSS que se pueden usar con ::first-letter incluyen:

  • Todas las propiedades de fuente.
  • Todas las propiedades de fondo.
  • Todas las propiedades de margen.
  • Todas las propiedades de relleno (padding).
  • Todas las propiedades de borde.
  • Propiedades como color, decoración de texto, sombra de texto, transformación de texto, espaciado entre letras y palabras, altura de línea, decoración de texto (color, línea, estilo), sombra de caja, flotación y alineación vertical.

Sintaxis

::first-letter {
  /* propiedades CSS */
}

Ejemplos:

En este ejemplo, la primera letra del párrafo (<p>) se mostrará un 200% más grande que el resto del texto y en color rojo. Este estilo se aplica solo a la primera letra del párrafo que sigue inmediatamente después de un elemento <h2>. 
p::first-letter {
  font-size: 200%;
  color: red;
}

::first-line

El pseudoelemento ::first-line de CSS se utiliza para aplicar un estilo único a la primera línea de un bloque o un elemento en línea siempre que ese elemento esté dispuesto como un bloque (es decir, no en línea o en cajas flexibles). Es especialmente útil para dar formato a la primera línea de un párrafo para captar la atención o simplemente para aplicar un estilo tipográfico.

Sintaxis

selector::first-line {
  /* reglas de estilo */
}

Ejemplos:

Cambiar la fuente y el color de la primera línea de un párrafo: 
p::first-line {
  font-weight: bold;
  color: blue;
}
Es importante señalar que ::first-line sólo puede aplicar ciertas propiedades CSS. Las propiedades que pueden ser aplicadas incluyen fuentes, colores, fondo, palabra y letra espaciado, y texto decoración entre otros. No se pueden aplicar propiedades como el borde, el margen, el padding o el display. Además, ::first-line sólo afecta al contenido que fluye normalmente dentro de un elemento; no afectará al contenido que ha sido absolutamente posicionado o flotado fuera del flujo normal del documento.

::placeholder

El pseudo-elemento ::placeholder en CSS se utiliza para aplicar estilos al texto provisional (placeholder) en un elemento <input> o <textarea>. Solo un subconjunto de las propiedades CSS que se aplican al pseudo-elemento ::first-line pueden ser utilizadas con ::placeholder

Sintaxis

::placeholder {
  /* propiedades CSS */
}

Ejemplos:

En este ejemplo, el texto provisional del campo <input> se mostrará en color rojo, con un tamaño de fuente del 120% del tamaño normal y en cursiva 
input::placeholder {
  color: red;
  font-size: 1.2em;
  font-style: italic;
}

::marker

El pseudo-elemento ::marker en CSS se utiliza para seleccionar la caja de marcadores de un elemento de lista, como <li> o <summary>, que generalmente contiene una viñeta o un número. Este pseudo-elemento es aplicable a cualquier elemento o pseudo-elemento configurado para display: list-item

Sintaxis

::marker {
  /* propiedades CSS */
}

Ejemplos:

En este ejemplo, los marcadores (::marker) de los elementos de la lista (<li>) en la lista desordenada ( <ul>) se muestran en color rojo y con un tamaño de fuente aumentado a 1.5 veces el tamaño normal 
ul li::marker {
  color: red;
  font-size: 1.5em;
}
Es importante señalar que ::first-line sólo puede aplicar ciertas propiedades CSS. Las propiedades que pueden ser aplicadas incluyen fuentes, colores, fondo, palabra y letra espaciado, y texto decoración entre otros. No se pueden aplicar propiedades como el borde, el margen, el padding o el display. Además, ::first-line sólo afecta al contenido que fluye normalmente dentro de un elemento; no afectará al contenido que ha sido absolutamente posicionado o flotado fuera del flujo normal del documento.

::backdrop

El pseudo-elemento ::backdrop en CSS se utiliza para crear una caja detrás de un elemento en la pila de la capa superior. Esta caja se muestra inmediatamente debajo del elemento en la capa superior, sirviendo como un fondo que puede ocultar el contenido subyacente. Es comúnmente usado para elementos que se muestran en pantalla completa.

Sintaxis

::backdrop {
  /* propiedades CSS */
}

Ejemplos:

En este ejemplo, el ::backdrop aplicará un fondo negro con una opacidad del 50% detrás del elemento de la capa superior, como un modal a pantalla completa, creando un efecto de oscurecimiento del contenido subyacente. 
::backdrop {
  background-color: rgba(0, 0, 0, 0.5);
}

::grammar-error

El pseudoelemento ::grammar-error de CSS se utiliza para aplicar estilos a partes del texto que han sido identificadas como errores gramaticales por la función de revisión gramatical del navegador. Esto es útil para resaltar y proporcionar retroalimentación visual sobre posibles errores en texto editable, como en un campo de formulario o en un editor de texto en línea.

Sintaxis

::grammar-error {
  /* reglas de estilo */
}

Ejemplos:

::grammar-error {
  color: red;
  text-decoration: underline wavy red;
}
Este ejemplo cambiará el color de texto de los errores gramaticales a rojo y los subrayará con una línea ondulada roja, típicamente utilizada para indicar errores.

::selection

El pseudoelemento ::selection de CSS se utiliza para cambiar el estilo de la porción de un documento que ha sido seleccionada por el usuario (por ejemplo, texto que el usuario ha resaltado con el ratón). Este estilo puede ser útil para hacer la selección más visible o para hacerla coincidir con el esquema de color de tu sitio web o aplicación.

Sintaxis

::selection {
  /* reglas de estilo */
}

Ejemplos:

::selection {
  background: blue;
  color: white;
}
Este código CSS cambiará el fondo de cualquier texto seleccionado a azul y el texto mismo a blanco.

::spelling-error

El pseudoelemento ::spelling-error en CSS se utiliza para aplicar estilos personalizados a fragmentos de texto que han sido identificados como errores ortográficos por la corrección ortográfica automática del navegador. Es similar al pseudoelemento ::grammar-error, pero se centra específicamente en errores de ortografía.

Sintaxis

::spelling-error {
  /* reglas de estilo */
}

Ejemplos:

::spelling-error {
  color: red;
  text-decoration: underline;
}
Este código cambiará el color del texto identificado como un error ortográfico a rojo y lo subrayará, lo que facilita su identificación visual por parte del usuario.
